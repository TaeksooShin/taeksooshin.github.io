DDL

  2023-02-17       데이터베이스(database) 

용어 설명

나의 해석

DDL(Data Defintion Language)은 데이터베이스 및 테이블을 생성, 삭제 하기 위해 사용하는 SQL문이다.

특징으로는 WHERE문을 사용 할 수 없다.

각종 설명

DDL(데이터 정의 언어)은 데이터를 저장하는 데이터베이스 및 테이블을 생성 삭제하기 위한 것이다.

(SQL 더 쉽게 더 깊게 p.15)

간략 설명

CREATE : 생성 DROP : 삭제 ALTER : 변경

상세 설명

CREATE

CREATE문은 데이터베이스, 테이블을 작성할 때 사용하는 [[SQL]]문이다. 기본 구문은 아래와 같다.

데이터베이스를 생성하는 CREATE DATABASE문

CREATE DATABASE <데이터베이스명>

테이블을 생성하는 CREATE TABLE문

CREATE TABLE <테이블>
(<열명1> <데이터형> <열의 제약>,
 <열명2> <데이터형> <열의 제약>,
 <열명3> <데이터형> <열의 제약>,
 ...
 <테이블의 제약1>,
 <테이블의 제약2>,
 <테이블의 제약3>,
 ...);

DROP

DROP문은 테이블을 삭제 할 때 사용하는 [[SQL]]문이다. 기본 구문은 아래와 같다.

테이블을 삭제하는 DROP TABLE문

DROP TABLE <테이블명>;

명심해야하는 점은 ‘삭제한 데이터베이스, 테이블은 복구할 수 없다.’ 반드시 조심해야한다.

ALTER

ALTER문은 테이블의 정의를 변경하는 사용하는 [[SQL]]문이다.

열 추가

테이블의 열을 추가할 때 사용하는 ALTER TABLE문

ALTER TABLE <테이블명> ADD COLUMN <열 정의>;

주의) Oracle과 SQL Server의 열을 추가하는 경우 COLUMN을 빼고 작성한다.

Oracle과 SQL Server에서의 테이블의 열을 추가할 때 사용하는 ALTER TABLE문

ALTER TABLE <테이블명> ADD <열 정의>;

열 삭제

테이블의 열을 삭제하는 ALTER TABLE 문

ALTER TABLE <테이블명> DROP COLUMN <열명>;

주의) Oracle에서 열을 삭제하는 경우 COLUMN을 빼고 작성한다.

Oracle에서 테이블의 열을 삭제하는 ALTER TABLE 문

ALTER TABLE <테이블명> DROP <열명>;

테이블 이름 변경

테이블의 이름 변경은 표준 SQL이 존재하지 않기 때문에 각 DBMS 마다 구문이 다르다. 각각의 구문은 아래와 같다.

Oracle, PostgreSQL

ALTER TABLE <대상 테이블> RENAME TO <변경하고 싶은 이름>;

DB2

RENAME TABLE <대상 테이블> TO <변경하고 싶은 이름>;

SQL Server

sp_rename '<대상 테이블>', '<변경하고 싶은 이름>';

MySQL

RENAME TABLE <대상 테이블> TO <변경하고 싶은 이름>;
