임포트(import)

  2023-03-13       프로그래밍 언어(programming language) 

[[java]]{자바(Java)}

사용 이유

다른 패키지에 속하는 클래스를 사용하기 위해 import를 이용한다.

위치

import문은 pacakge 선언과 class 선언의 사이에 위치한다.

* (에스터리스크)

패키지에 포함된 다수의 클래스를 사용해야 한다면 클래스별로 import문을 작성할 필요 없이 클래스 이름을 생략하고 * 를 사용해서 표현 할 수도 있다. 단 개인적인 의견으로는 static 클래스를 제외하고는 import문은 * 로 표현하기 보다는 사용할 클래스를 하나씩, import문을 이용해서 작성하는 것이 좋다고 생각한다. 왜냐하면 * 로 표현되어 있는 경우, 현재의 [[class]]{클래스(class)}와 다른 [[class]]{클래스(class)}간의 [[coupling]]{결합도(coupling)}가 얼마나 높은지 판단 할 수 없다. 그렇기 때문에 [[refactoring]]{리팩터링(refactoring)}의 판단 기준이 될 수 있는 [[coupling]]{결합도(coupling)}가 눈에 보이지 않게 된다. 결과적으로는 코드가 점점 냄새나는 코드로 변화한다.
