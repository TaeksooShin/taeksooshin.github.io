bundler

  Jan 24, 2022       ruby 

Install

자신의 터미널에서 아래의 커맨드를 입력한다.

gem install bundler

(만약 gem 커맨드를 사용 할 수 없다면 rubygem을 참고해서 설치 할 것)

설치 후 아래의 커맨드를 이용해서 설치가 되었는지 확인 할 것

bundler -v

How to use

  1. 번들러 init bundler를 사용하고자 하는 디렉토리에서 아래의 커맨드를 입력한다.
bundle init
  1. gem 파일 기록 그 후 사용하고자 하는 gemfile 목록을 목적에 맞게 Gemfile 내부에 기록할 것

목적 1. 단순히 gem을 원하는 경우

gem 'rubocop'

목적 2. application에서 자동으로 require가 되지 않도록 하기 위한 설정을 추가하는 경우

gem 'rubocop', require: false

rubocop의 경우는 require가 될 필요가 없으므로(CLI 환경에서 필요한 도구이다.) require: false를 넣어주는 것이 타당하다.
참고: rubocopをbundlerでインストールするときにrequire: falseにする理由 - Qiita ruby - Why does rubocop recommend require: false in Gemfile? - Stack Overflow

목적 3. 특정 버전을 지정하는 경우

gem 'rubocop', '~> 1.8', require: false

대부분의 경우 위와 같이 사용할 것

  1. gemfile 인스톨 기록한 gem파일을 설치하기 위해서는 아래의 커맨드를 입력해 준다.
bundle install --path {path_name}

gem 파일을 인스톨할 장소를 {path_name}에 기록해 주면 ok
gem 파일 리스트를 변경 하고 인스톨 할 때마다 path를 기록할 필요없이 bundle install을 입력하면 처음에 입력한 path에 자동으로 인스톨 해준다.
