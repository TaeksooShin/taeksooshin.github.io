mac - 원화 표시(₩)를 백쿼트(`)로 변경하기

  2022-03-21       mac 

개발자라면 한글로 타자를 치다가 백쿼트(`)를 입력했을 때 원화 표시(₩)가 튀어나와서 원화 표시를 지우고 영문으로 변경 후에 다시 백쿼트를 입력한 경험이 매우 많을 것이다. 한두 번은 괜찮지만 이러한 상황이 몇 번이고 반복된다면 맥 OS를 지급한 회사를 폭파하던가 리눅스로 작업 가능한 다른 회사로 전근 갈지 고민을 하기 시작한다. 언제나 성급한 결정은 금물이다.

아래의 방법을 사용해보고 회사를 폭파할지 말지, 다른 회사로 전근 갈지를 결정하도록 하자.

한글 키보드에서 원화 표시를 백쿼트로 변경하기

DefaultkeyBinding.dict에 원화의 입력을 받았을 때 백쿼트를 입력하도록 변경하면 된다.

수동으로 변경하기

  1. ~/Library 디렉토리로 이동한다.
  2. Library 디렉토리 아래 KeyBindins 디렉토리를 생성한다.
  3. DefaultkeyBinding.dict 파일을 생성한다.
  4. DefaultkeyBinding.dict 파일에 아래의 코드를 추가한다.
{
  "₩" = ("insertText:", "`");
}
  1. 다시 시작한 뒤에 한글 키보드를 변경 후 백쿼트 입력으로 변경되어 있는지 확인해본다.

shell script를 이용해서 변경하기

아래의 스크립트 사용하자.

#!/bin/sh

if [ -f ~/Library/KeyBindings/DefaultkeyBinding.dict ]; then
	echo "~/Library/KeyBindings/DefaultkeyBinding.dict가 이미 존재합니다."
	exit -1
fi

mkdir -p ~/Library/KeyBindings
touch ~/Library/KeyBindings/DefaultkeyBinding.dict
cat << EOF > ~/Library/KeyBindings/DefaultkeyBinding.dict 
{
  "₩" = ("insertText:", "\`");
}
EOF

echo "완료했습니다."

결론

오늘도 회사를 무사하게 지켜낸 코더 신씨는 누가 원화 표시를 키보드에 넣었는지, 대체 무슨 생각으로 넣었는지에 대해서 의문을 품으며 자신의 잘못된 선택과 디자인이 사용자들에게 얼마나 큰 피해를 줄 수 있는가에 대해, 뼛속 깊이 느끼며 하루를 마무리했다.
